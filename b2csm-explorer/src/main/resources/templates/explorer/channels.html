Channel 1 - b2csm-honeypots
Proxy-based CSM queries: we use a dataset collected from a set of honeypots spanning over 7000 IP addresses. To increase network density, we limit our analysis to only the most active subnet within the dataset.
Channel 2 - b2csm-nids
NIDS-centered CSM: to demonstrate the NIDS-centered CSM, we conduct a case study using data from the 2017 CDX Competition, as collected by the USMA team.
Channel 3 - b2csm-gtmw
URL-centered CSM: we use the Georgia Tech data as if it were collected as such. The GT data contains mappings between applications, which are malware instances in this particular dataset.
